## Project Overview

Lunula is a hair and beauty product ecommerce store built on **Shopify Dawn v10.0.0** (Online Store 2.0), with light customizations on top of the base theme. The goal is to stay on Liquid and optimize the existing Liquid foundation — **do not migrate to Hydrogen/Oxygen**. Reasoning: the store's `@app` block structure and Judge.me review integration are not viable to migrate headlessly without significant effort and SEO risk. However, in the future, the goal may change to migrate to Hydrogen/Oxygen, so we do not want to make any big changes to make a migration more difficult. 

The repo is deployed to Shopify via the Shopify CLI. There is no build step — files are served directly by Shopify's CDN.

## Shopify CLI Commands

```bash
# Authenticate (one-time)
shopify auth login

# Push theme changes to a development theme (safe — doesn't affect live store)
shopify theme push --development

# Pull current live theme to sync local files
shopify theme pull

# Start local development server with hot reload
shopify theme dev

# Check theme for errors
shopify theme check
```

`shopify theme dev` is the primary development workflow. It opens a browser preview that reflects local file changes in real time without pushing to the live store.

## Architecture

This is a standard **OS 2.0 JSON template** theme. The key mental model:

- `templates/*.json` — define which sections appear on each page and their initial block configurations. These are auto-generated by the Shopify admin; avoid hand-editing unless necessary.
- `sections/*.liquid` — the actual page-building blocks. Each section has Liquid markup + a `{% schema %}` block defining its settings UI. Sections render `{% for block in section.blocks %}` and handle `{% when '@app' %}` to slot in third-party app blocks.
- `snippets/*.liquid` — reusable partials rendered via `{% render 'snippet-name' %}`. Cannot directly access parent scope variables (must be passed explicitly).
- `layout/theme.liquid` — the global HTML shell. All pages render inside this.
- `assets/` — flat directory of JS, CSS, and static files. No bundler. Scripts are loaded with `defer` via `asset_url` filter.
- `config/settings_data.json` — live theme settings (auto-generated by Shopify admin; treat as read-only).
- `config/settings_schema.json` — defines the Theme Settings UI panels in the Shopify admin.
- `.shopify/metafields.json` — currently empty; no custom metafields defined yet.

## Custom Code Added on Top of Dawn

These are the only meaningful departures from stock Dawn v10:

### 1. Time-based Dark Mode (`layout/theme.liquid`, lines 322–350)
A `DOMContentLoaded` JS block that applies `.dark-mode` to `<body>` based on hour of day (7pm–7am) and stores user preference in `localStorage`. **Known issue:** this causes a flash of unstyled content (FOUC) on page load because it runs client-side after render. It also won't respect user preference on first SSR paint. Needs to be replaced with an inline `<script>` in `<head>` (before CSS) or a cookie-based approach.

### 2. Scroll Animations (`assets/animations.js`)
Uses `IntersectionObserver` to add/remove `scroll-trigger--offscreen` class. Conditionally loaded based on `settings.animations_reveal_on_scroll`. Works correctly; no known issues.

### 3. Blob/Mask Shapes (`assets/mask-blobs.css`, `snippets/mask-arch.liquid`)
Custom CSS shapes used for image masking. Decorative only.

### 4. Product Card — Full Card Link (`README.md` v1.0.1)
The entire product card was made a clickable link. See `snippets/card-product.liquid` for implementation.

## App Integration Pattern

Third-party apps (Judge.me, payment processors, etc.) are integrated via the standard OS 2.0 `@app` block system — **not** via hardcoded script tags. This means:

- Apps inject through `{{ content_for_header }}` in `layout/theme.liquid`
- App blocks appear in sections that include `{% when '@app' %} {% render block %}` — currently: `header.liquid`, `main-product.liquid`, `featured-product.liquid`, `footer.liquid`, `main-cart-footer.liquid`, `newsletter.liquid`, `main-article.liquid`
- No Klaviyo, Judge.me, or other app code is hardcoded in Liquid files

**Judge.me reviews** are connected but rendered via app blocks on the product page. The `show_rating: false` flag in `templates/product.json` means star ratings are currently hidden on the product template — this may be intentional (pending Judge.me block placement) or an oversight worth checking.

## Known Issues / Priority Optimization Work

1. **Dark mode FOUC** — replace the `DOMContentLoaded` dark mode script with an inline `<head>` script to prevent flash
2. **`spacing_sections: 0`** in `config/settings_data.json` — all section spacing is zeroed out; verify this is intentional
3. **No metafields defined** — if product-level custom data (e.g., CBD potency, certifications) is needed, metafields should be defined in `.shopify/metafields.json`
4. **`show_rating: false`** on product template — Judge.me ratings may not be surfacing on PDPs; verify intent
5. **CSS per-section loading** — `main-product.liquid` loads ~7 CSS files inline via `stylesheet_tag`. This is Dawn's default pattern and generally fine, but worth auditing if LCP scores are poor

## What Not to Change

- `config/settings_data.json` — managed by Shopify admin; manual edits get overwritten
- `templates/*.json` — auto-generated; prefer editing via Shopify admin theme editor
- Dawn core JS/CSS that has no custom modifications — track upstream Dawn changes instead of forking logic
